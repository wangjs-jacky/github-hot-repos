# JustVugg/colibri

Run GLM-5.2 (744B MoE) on a 25GB-RAM consumer machine — pure C, zero deps, experts streamed from disk. Tiny engine, immense model. 🐦

项目定位  
一个在消费级设备上运行超大规模 GLM-5.2 模型的 C 语言实现，通过零依赖和磁盘流式加载专家模型，解决资源受限环境下的大模型部署问题。

核心功能  
- 在 25GB 内存的机器上运行 744B 参数的 GLM-5.2 模型  
- 使用纯 C 语言实现，无外部依赖  
- 从磁盘流式加载模型专家，减少内存占用  
- 支持高效推理和模型调用  

技术特点  
采用轻量级 C 语言实现，通过磁盘流式加载技术降低内存需求，确保在消费级硬件上高效运行大型模型，具备高度优化的性能和可移植性。

适用场景  
- 个人开发者在本地测试大模型  
- 资源受限的嵌入式系统部署  
- 研究人员进行模型推理实验  
- 需要低延迟、高效率的推理任务  

推荐理由  
适合希望在普通硬件上运行超大规模模型的用户，提供高性能、低资源消耗的解决方案，是大模型落地的重要工具。
